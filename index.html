Histórico

Nesta Seção será apresentado um breve histórico da ferramenta. O Nagios foi desenvolvido e lançado em 1999, por Ethan Galstad, como um projeto open source, inicialmente utilizando o nome NetSaint.Em 2002, devido para evitar problemas de direitos de marca, mudou o nome para Nagios (Acrônimo recursivo para Nagios Aint Gonna Insist On Sainthood, em tradução livre, Nagios não vai insistir em Santidade, trocadilho com o primeiro nome da ferramenta, NetSaint). Em 2009, ganhou o prêmio da Linux Journal Reader's Choice de "Aplicação Linux de monitoramento favorita". Ainda em 2009, a ferramenta foi baixada mais de 600.000 vezes no sourceforge.net

Visão Geral

O Nagios é ferramenta de gerenciamento de rede que permite que organizações identifiquem e resolvam problemas de infraestrutura de TI, antes que eles afetem criticamente o processo de negócio" Monitora a infraestrutura de IT para garantir que sistemas, aplicações e serviços estejam funcionando devidamente. Em caso de falhas, o Nagios envia alertas para que elas sejam resolvidas antes afetem processos, usuários ou clientes. Esses alertas podem ser enviados atráves de SMS/Email.
Utlizando o Nagios é possível:

  • Planejar atualizações antes que sistemas desatualizados causem falhas (Ligado a àrea de gerência de configuração do OSI
  • Consertar problemas automaticamente quando eles são detectados;
  • Garantir que indisponibilidades de infraestrutura (outages) tenham um efeito mínimo na sua organização;
  • Monitorar toda a infraestrutura e processos de negócio.

Funcionalidades e vantagens

Monitoramento:
  • Monitora aplicações, serviços, sistemas operacionais, protocolos de rede, métricas de sistema e componentes de infraestrutura utilizando uma única ferramenta;
  • Poderosas APIs de script que permitem fácil monitoração.

Visibilidade
  • Visão centralizada de toda a infraestrutura de TI monitorada;
  • Detalhada informação de status disponível através de interface web.

Consciência
  • Rápida detecção de indisponibilidade de infraestrutura;
  • Alertas podem ser enviados para os administradores de rede através de email ou SMS;
  • Capacidades de escalonamento garantem que as notificações cheguem nas pessoas certas.

Remediação de problemas.
  • "Event handlers": Esta funcionalidade permite a criação de script que são executados quando algum evento ocorre. Exemplo: Imagine que você tenha uma servidor apache e que ele está sendo monitorando através do Nagios. No momento em que esse servidor cair é possível criar um event handler para que, de forma automática, um script levante o serviço do apache novamente.

Arquitetura expandível
  • Integração entre aplicações locais e de terceiros através de diversas APIs;
  • Centenas de plugins> e addons desenvolvidos pela comunidade que extendem o funcionamento do Nagios Core

Estável, Confiável e Comunidade ativa.
  • Mais de 10 anos de desenvolvimento ativo;
  • Escalável para monitorar milhares de nós;
  • Estimados 1 milhão de usuários pelo mundo;
  • Lista de emails da comunidade provem suporte de graça.

Código reutilizável.
  • Open source software;
  • Acesso total ao código fonte;
  • Lançado através da licença GPL.

Instalação

O Nagios pode ser instalado em sistemas Linux (Ou Unix). O Site oficial fornece tutoriais para instalação nos seguintes sistemas: Fedora, Ubuntu e openSUSE. Este documento irá apresentar a instalação do Nagios no Fedora. Para um tutorial de instalação nos outros sistemas, clique aqui para instalar o nagios no ubuntu e aqui para instalar no openSUSE.


Instalação no Fedora:

Para instalar o Nagios no Fedora é preciso ter acesso root e instalar os seguintes pacotes: apache, php gcc e a biblioteca gb. Para Fazer isso, execute os comandos: 

# yum install -y httpd php gcc glibc glibc-common gd gd-devel

Crie um usuário "nagios" e dê um password a ele:

# useradd -m nagios
# passwd nagios

Criar o grupo nagcmd (permite que comandos externos sejam enviados pela interface web. Adicionar os usuários nagios e apache ao grupo:

# grupoadd -m nagcmd
# usermod -a -G nagcmd nagios
# usermod -a -G nagcmd apache

Baixe os arquivos do Nagios Core e Nagios Plugin

# wget http://prdownloads.sourceforge.net/sourceforge/nagios/nagios-3.2.3.tar.gz
# wget http://prdownloads.sourceforge.net/sourceforge/nagiosplug/nagios-plugins-1.4.11.tar.gz

Compilar e instalar o Nagios Core

# tar xzf nagios-3.2.3.tar.gz
# cd nagios-3.2.3
# ./configure --with-command-group=nagcmd
# make all
# make install
# make install-init
# make install-config
# make install-commandmode

Altere o arquivo /usr/local/nagios/etc/objects/contacts.cfg para configurar o email no qual você deseja receber os alertas

Instale o arquivo de configuração da interface web no diretório do apache, crie um usuário para se logar na interface web e reinicie o Apache:

# make install-webconf
# htpasswd -c /usr/local/nagios/etc/htpasswd.users nagiosadmin
# service httpd restart

Compilar e instalar o Nagios Plugins:

#tar xzf nagios-plugins-1.4.11.tar.gz
# cd nagios-plugins-1.4.11
# ./configure --with-nagios-user=nagios --with-nagios-group=nagios
# make
# make install

Fazer com que o Nagios inicie automaticamente no sistema:

# chkconfig -add nagios
# chckconfig service nagios on
# service nagios start

Por fim, abra o navegador e se logue na interface web:

http://localhost/nagios
